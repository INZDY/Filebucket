# Filebucket MVP Plan

## Goal
- Build Filebucket as a personal hosted file-and-note vault.
- Keep the MVP private and single-user first.
- Use folders, Markdown notes, tags, filename/title search, trash/restore, inline image support, and backup/export.
- Keep collaboration, sharing, comments, backlinks, teams, and block-editor features out of scope unless explicitly requested.

## Architecture
- Next.js App Router is the app and backend.
- Auth.js handles login and session helpers.
- Prisma manages relational data.
- Supabase PostgreSQL is the hosted database.
- Cloudflare R2 stores media blobs.
- Browser uploads to R2 should use short-lived presigned URLs generated by Next.js.
- PostgreSQL stores users, folders, notes, tags, trash state, media metadata, and backup/export metadata.
